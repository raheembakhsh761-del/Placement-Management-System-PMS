# üöÄ Namal Placement Management System (NPMS)

The **Namal Placement Management System (NPMS)** is a **Problem Based Learning (PBL)** project developed for the **Data Structures & Algorithms** course at **Namal University**, under the supervision of **Sir Farrukh Qureshi**.

This system provides a **centralized, web-based platform** to manage placement-related activities such as job postings, student records, and opportunity notifications using efficient **Data Structures and Algorithms**.

---

## üìå Project Overview

Effective communication between placement offices and students is critical for sharing information about:

- Internships  
- Job opportunities  
- Exchange programs  
- Competitions  

**NPMS** solves this problem by offering a **single, organized portal** where verified opportunities are shared in real time.

---

## ‚ùì Problem Statement

Traditional placement management suffers from:

- Unorganized data  
- Multiple communication platforms  
- Lack of real-time updates  

**NPMS** addresses these challenges using **optimized data structures and algorithms** for fast data storage, retrieval, and processing.

---

## üß© Core Modules Implemented

### üîπ Module A ‚Äì Priority Processor
- **Purpose:** Prioritize students based on CGPA  
- **Data Structure:** Min-Heap / Sorting Algorithms  
- **Function:** Efficient retrieval of high-priority students  

### üîπ Module B ‚Äì Rapid Registry
- **Purpose:** Instant lookup of students and opportunities  
- **Data Structure:** Hash Tables (Python Dictionaries)  
- **Function:** O(1) average-time search for authentication and records  

### üîπ Module C ‚Äì Connectivity Network
- **Purpose:** Calculate distances between cities  
- **Algorithm:** Dijkstra‚Äôs Shortest Path Algorithm  
- **Function:** Finds shortest paths in weighted graphs  

### üîπ Module D ‚Äì Organized Archive
- **Purpose:** Maintain sorted student records  
- **Data Structure:** Binary Search Tree (BST)  
- **Function:** Efficient insertions, deletions, and traversal

## üåê Key Features

  - Students

## ‚öôÔ∏è Working Principle

1. Users log in according to their role  
2. Placement Officers register students and post opportunities
